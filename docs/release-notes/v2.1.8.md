# v2.1.8

## 版本概览

`v2.1.8` 是 `v2.1.7` 的紧急修复版本，专门解决移动端浏览器无法滚动整页内容的问题。强烈建议所有 v2.1.7 用户升级。

## 修复

### 移动端整页滚动失效

- v2.1.7 为优化页面切换动画，给路由容器 `.route-shell` 加了 `contain: layout style`，使其成为新的块级格式化上下文（BFC）。
- 配合 `flex: 1 + min-height: 0` 的设定，导致路由容器在移动端 column flex 容器中**无法被子内容撑高**——子页面通过 `overflow: visible` 渲染到容器外，但 box 高度永远等于父容器可视高度，外层 `.layout-main` 看不到滚动需求，整页就无法滚动。
- 桌面端依赖 `.dd-scroll-page` 自身 `overflow: auto` 滚动，所以桌面端没受影响；移动端没有这个内嵌滚动容器，只依赖 `.layout-main` 的 native scroll，因此被卡死。

### 修复方案

- 移动端 `.route-shell` 改为 `flex: 1 0 auto`（禁止收缩），并显式 `contain: none`、`overflow: visible`，让子内容能撑高路由容器。
- 子页面 `.dd-fixed-page` / `.dd-scroll-page` 在移动端同样改为 `flex: 1 0 auto + overflow: visible`，确保 box 高度等于内容高度。
- `.layout-main` 加 `-webkit-overflow-scrolling: touch`，让 iOS 上滚动惯性更顺滑。
- 桌面端行为完全不变：仍然是 `flex: 1 + min-height: 0 + contain: layout style + overflow: hidden`，依赖 `.dd-scroll-page` 内嵌滚动。

## 测试与验证

- `pnpm exec vue-tsc --noEmit` 类型检查通过
- `pnpm run build` 构建通过
- 手动验证：iPhone Safari / Android Chrome 各页面整页滚动恢复正常；桌面端动画 / 滚动 / 暗黑模式色差未受影响

## 升级说明

- 推送 Git tag `v2.1.8` 后会触发 GitHub Release 工作流，产出 Docker / 二进制 / Magisk ZIP 与 `update.json`。
- 已装 `v2.1.7` 的设备：
  - **Docker / 二进制**：面板内点「检查系统更新」即可一键升级。
  - **Magisk 模块**：模块卡片自动收到 `v2.1.8` 更新提示（versionCode=20108）。
- 仅修复样式层，无需任何配置 / 数据迁移。
