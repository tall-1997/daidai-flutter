## 新功能

### 用户头像上传

- 个人设置页支持上传自定义头像，支持 JPG、PNG、GIF、WebP 格式，最大 2MB。
- 头像区域 hover 显示相机图标，点击即可选择文件上传；已有头像时右下角出现删除按钮。
- 顶栏用户下拉菜单优先显示头像图片，未设置头像时回退为用户名首字母。
- 后端新增 `POST /auth/avatar`（上传）、`DELETE /auth/avatar`（删除）、`GET /auth/avatar/:filename`（静态服务）三个端点，头像以 UUID 命名存储在 `data/avatars/` 目录，上传新头像时自动清理旧文件。

### 任务视图管理

- 定时任务页新增视图管理功能，点击 ⚙ 按钮打开管理对话框。
- 支持隐藏不常用的视图标签（「全部」视图不可隐藏），支持拖拽排序自定义视图顺序。
- 视图的隐藏和排序状态持久化到数据库，刷新页面不丢失。

### 机器码

- 系统启动时自动生成标准 UUID v4 大写格式的机器码（形如 `A1B2C3D4-E5F6-4789-ABCD-1234567890AB`）并持久化到数据库，重启不变。旧格式（8 段 hex）数据在首次启动时会自动迁移为新格式。
- 系统设置页「系统信息」卡片新增机器码显示行，支持一键复制。

### 任务成功通知附带日志摘要

- 任务执行成功的通知消息现在会附带最后若干行脚本输出日志，方便在通知渠道中直接查看执行结果，无需再跳转到面板查看。
- 自动过滤面板内部元数据行（如依赖安装日志），只保留脚本自身输出。

## 安全增强

### 双因素认证 (2FA) 安全加固

- **禁用 2FA 需要验证动态码**：禁用操作前必须输入当前认证器的 6 位动态验证码，防止会话被劫持后被攻击者关闭 2FA。后端 `DELETE /security/2fa` 端点新增 `code` 参数校验。
- **登录 2FA 改为弹窗输入**：当后端返回 `two_factor_required` 时，前端弹出独立的 TOTP 验证码对话框，用户输入 6 位数字后自动提交，体验更清晰。

## 修复

### Python 依赖自动安装误报

- 修复 `pysmx`、`socks` 等模块反复被误报为缺失并循环安装的问题。根因是 `importlib.import_module()` 会执行模块代码，部分模块在导入时因缺少运行时环境而抛出非 `ImportError` 异常（如 `socks` 模块在导入时触发内部错误），导致被判定为未安装。现改用 `importlib.util.find_spec()` 仅检查文件是否存在，不执行模块代码。
- 扩展模块名→PyPI 包名的别名映射表，新增 `socks→pysocks`、`cv2→opencv-python`、`Crypto→pycryptodome`、`yaml→pyyaml`、`git→gitpython`、`dotenv→python-dotenv` 等 12 个常见映射，避免安装错误的包。
- 新增 `installedDeps` 去重机制，同一依赖在一次任务执行中不会重复安装。

### 环境变量更新身份识别

- 环境变量的业务身份统一改为 `(变量名, 备注)` 二元组。创建（POST）改为 upsert 语义：相同 `(name, remarks)` 的变量覆盖更新而非报错；更新（PUT）支持修改备注和启用状态，同时检测目标身份冲突和无变化跳过；导入合并也按 `(name, remarks)` 匹配。
- 数据库层新增 `(name, remarks)` 唯一索引（已有重复数据的老库会跳过索引创建，应用层校验仍然生效）。
