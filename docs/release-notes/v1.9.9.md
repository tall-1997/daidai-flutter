# v1.9.9

## 更新内容

### 脚本运行与调试

- 重构脚本运行时启动方式，不再把超长环境变量直接塞进进程参数，修复长变量内容导致的 `argument list too long` 启动失败。
- Python、Node.js、TypeScript 调试与执行增加运行时可执行文件兜底解析，修复部分环境下 `node: no such file or directory`、Python 虚拟环境路径失效等启动问题。
- 调试运行改为默认不自动保存，未点击保存时只使用临时文件执行，避免调试过程中误覆盖正式脚本内容。
- 脚本内容/上传上限提升到 `100MB`，缓解大体积加密脚本无法运行或上传的问题。

### 脚本管理体验

- 修复 `.so` 等二进制脚本文件下载异常，下载逻辑改为走后端真实文件流，不再误用编辑器文本内容。
- 版本历史对比能力进一步增强，修复部分场景下无法显示差异的问题，并支持空白差异忽略、移动端纵向对比与无差异空状态提示。
- 版本历史弹窗放大，展示更多版本信息，同时新增“清空版本历史”按钮，可一键清空当前脚本的历史记录。

### 备份与恢复

- 备份恢复上传取消前端 `30` 秒超时限制，避免大备份包上传较慢时被直接判定失败。
- 创建备份时支持自定义备份包名称，不再只能使用随机生成名称。
- 日志内容上限默认值提升到 `102400000`，并为旧配置补充默认值迁移，升级后无需手动修正。

### 环境变量

- 环境变量页面新增“复制变量”按钮，可快速复制同名变量并重新填写值。

### 系统更新

- 系统更新会自动识别当前运行镜像来源：Debian 运行时走 `debian` 更新通道，Alpine 运行时走 `latest` 通道，避免 Debian 用户被错误切回 Alpine 镜像。
- 检测到新版本后，面板内会直接展示目标版本号和更新日志，不再需要手动跳转 GitHub 查看。
- 新增静默更新开关。开启后每 `24h` 自动检测更新，发现新版本会自动执行更新，并在成功或失败后通过通知渠道推送结果。
