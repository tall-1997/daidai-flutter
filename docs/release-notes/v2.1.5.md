# v2.1.5

## 版本概览

`v2.1.5` 是 `v2.1.4` UI 界面重构后的体验修复版本，重点处理桌面端页面固定布局、个人头像、环境变量分页、系统健康检查记录、订阅拉取冲突误判、依赖管理操作区、脚本管理窄屏布局，以及日志 ANSI 转义码显示等问题。

## 体验优化

### 页面布局与桌面端滚动

- 桌面端核心页面统一接入固定页面容器，避免整页出现不必要的上下滚动。
- 保留移动端和内容密集页的内部滚动能力，列表、表格和日志区域在页面内部滚动，不再把主页面撑高。
- 脚本管理在窄桌面宽度下切换为紧凑布局，避免文件树和编辑器互相挤压导致编辑区只剩很窄一列。

### 依赖管理

- 依赖列表压缩名称列和图标尺寸，依赖名过长时单行省略并保留悬停提示。
- 表格操作区改为“详情 / 重装或取消 / 更多”，卸载与强制卸载收进更多菜单，解决右侧操作显示不全的问题。
- Android 运行时安装日志和依赖安装日志现在支持 ANSI 彩色输出。

### 日志显示

- 执行日志详情、日志文件查看、依赖安装日志、Android 运行时安装日志、订阅历史日志和订阅实时拉取日志均支持 ANSI SGR 转义码显示。
- ANSI 渲染支持基础颜色、加粗、斜体、下划线、256 色和 RGB 真彩色。
- 清理日志里的清屏、光标移动等不适合网页展示的控制序列，减少乱码残留。
- 日志内容会先进行 HTML 转义，再转换为受控的样式片段，避免直接渲染原始日志 HTML。

## 功能修复

### 个人设置头像

- 修复上传头像后前端访问头像地址可能被 Nginx 静态图片规则抢走，导致 `/api/v1/auth/avatar/...` 返回 404 的问题。
- 头像加载失败时自动回退为默认头像，避免个人设置页显示破图。

### 环境变量

- 环境变量页面补回每页数量选择，支持在分页器里切换 page size，不再固定只能一页显示 20 条。

### 系统设置

- 系统健康检查结果改为持久化快照，刷新页面后仍能看到上次检查结果。
- 系统设置页进入时会读取最近一次健康检查记录，并正确显示“上次检查”状态。

### 订阅管理

- 修复“保留本地修改”拉取模式在没有本地改动时仍尝试恢复 stash，导致“未发现贮藏条目”并被误报为冲突的问题。
- 订阅拉取前改用 `git status --porcelain --untracked-files=all` 判断是否真的存在本地修改，只有需要时才创建 stash。
- stash 恢复流程改为明确的 `git stash push --include-untracked`，减少误判和异常提示。

## 兼容性与安全

- Docker 内置 Nginx 的 `/api/` 代理规则优先级提升，避免 API 路径被静态资源规则错误命中。
- 日志 ANSI 渲染只处理文本到安全 HTML 的展示层，不改变后端日志保存、复制和下载内容。
- 系统健康检查接口拆分为读取快照与立即检查，避免刷新页面丢失最新检查结果。

## 测试与验证

- 前端构建：`web/npm run build`
- 后端测试：`server/go test ./...`
- 新增系统健康检查持久化测试
- 新增订阅保留本地修改模式回归测试

## 升级说明

- 推送 Git tag `v2.1.5` 后会自动触发 GitHub Release 工作流。
