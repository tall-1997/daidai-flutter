# v2.2.10

## 修复

- **订阅拉取后不自动建任务**：脚本头没写 cron 注释时，旧版本会静默跳过整个仓库，一个任务都建不出来。v2.2.10 起业务脚本无论有没有 cron 头都会建任务——脚本有 cron 用脚本的；没有则用系统设置 `default_cron_rule`，未配置时用兜底"每天 0 点"。`sendNotify.js` / `notify.py` 等通知 helper 自动识别并跳过。
- **Python 自动安装依赖报 PEP 668**：v2.2.4 重构托管 venv 时漏了 `venvDir` 参数，venv 永远建不出来，自动安装 fallback 到系统 pip 撞 Alpine 的 `externally-managed-environment`。修回 venv 创建，并在系统 pip 路径加 `--break-system-packages --user` 兜底。
- **Dockerfile `ARG VERSION` 长期停留在 `2.0.4`**：构建出来的镜像版本号一直错，本次同步修正。

## 新增

- **Magisk 模块持久化更新备份**：每次更新前把 `daidai.db / scripts / backups / config.yaml / .jwt_secret / panel.log` 镜像一份到 `/data/adb/daidai-panel/last-update-backup/`（Magisk 持久化目录，模块更新不会清空）。原子切换防中途失败丢备份，跳过体积大的 `logs/` `deps/`。
- **一键恢复脚本**：`su -c "sh /data/adb/daidai-panel/restore-last-update.sh"` 自动找 rootfs、确认后停面板、覆盖回数据，无需手动 cp。

## 兼容性

升级即生效，无需任何手动操作。
