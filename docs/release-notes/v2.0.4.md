# v2.0.4

## 修复内容

### 定时任务与依赖

- 修复定时任务页每页条数刷新后总是回退到默认值的问题，用户选择的分页数量现在会被稳定记住。
- 修复任务视图筛选、标签识别、订阅标签显示和命令展示顺序异常，任务列表在筛选和排序场景下与实际数据保持一致。
- 修复脚本自动安装依赖后下次运行仍重复提示缺失的问题，并补齐 Python / Node.js / Linux 依赖状态在重启、恢复和更新后的收口逻辑。
- 新建任务时“失败时通知”默认改为关闭，避免批量创建任务时误触发不必要的告警。

### 更新、脚本与界面

- 修复面板更新后 Shell 依赖已完成更新却仍显示安装失败的问题，更新与重启后的依赖状态现在会正确校验和续跑。
- 修复脚本页面保存后立即下载仍拿到旧版本内容的问题，脚本下载链路现在会读取最新保存内容。
- 订阅拉库自动建任务时，若脚本内存在 `new Env(...)` 名称声明，会优先使用脚本声明名作为自动生成的任务名称。
- 后台日志背景图片配置已真正接入运行时展示，上传后的背景图可以在日志相关视图中即时生效。

### 性能与容器运维

- 任务列表默认查询改为数据库侧分页和排序，减少运行中任务轮询时的大量无效内存排序开销。
- 任务页、日志页、依赖页现在会在页面被 `keep-alive` 缓存或浏览器标签页隐藏时暂停轮询，降低后台空转占用。
- 仪表盘执行趋势图改为延迟初始化，只有图表区域进入视口后才加载图表模块，减轻首屏压力。
- 容器新增 `ddp` 内置命令，支持在终端中直接执行状态检查、日志查看、备份管理、任务运行、订阅拉取、环境变量管理和常用运维操作。
