# v2.1.4

## 版本概览

`v2.1.4` 是一次以 **UI 界面重构** 为核心的版本更新。此次重点不只是换皮，而是把仪表盘、定时任务、执行日志、环境变量、订阅管理、脚本管理、AI Code、系统设置、通知渠道、Open API、用户管理、个人设置等核心页面的布局、信息层级和移动端适配一起重做，并同步修复了一批前后端状态不一致、跳转失效、权限误展示和数据口径错误的问题。

## UI 界面重构

### 仪表盘

- 重做欢迎横幅、快捷操作、系统资源、最近活动、最近执行任务、任务统计等模块布局
- 优化统计卡片与图表信息层级，首屏更聚焦，桌面端和移动端显示更统一
- 快捷入口与详情入口重新梳理，减少“看得到但点不动”或“文案与行为不一致”的情况

### 定时任务 / 执行日志

- 重构任务列表与日志列表页面，统一状态卡、筛选栏、批量操作区和移动端卡片样式
- 强化运行状态、最后运行、下次运行、触发方式等信息展示，列表可读性更高
- 按角色收敛操作入口，避免 viewer 账户看到实际上无法执行的按钮

### 环境变量 / 订阅管理

- 环境变量页面重做统计区、搜索区、分组筛选与批量操作区
- 订阅管理页面重做顶部概览、订阅卡片、设置弹窗和批量删除体验
- 空状态、导入导出、批量改名/分组等入口更清晰

### 脚本管理 / AI Code

- 脚本管理工作区重新整理为更清晰的多面板布局，文件树、编辑区、调试与 AI 辅助流转更顺手
- AI Code 页面补齐独立工作区体验，可直接选择文件、生成脚本、调试当前脚本并回写结果
- 持续优化脚本编辑、AI 生成、调试日志、工作区浏览之间的联动

### 系统设置 / 通知渠道 / Open API / 用户相关页面

- 系统设置页拆分概览、安全、系统配置、告警、更新等卡片区域，信息组织更完整
- 通知渠道页改为展示真实配置概览、最近测试状态、今日发送统计，不再使用前端伪字段
- Open API 页面重做统计卡、趋势图、列表与移动端卡片，并修正文案口径
- 用户管理、个人设置、依赖管理等页面同步刷新样式与交互结构

## 功能修复

### 仪表盘与任务链路

- 修复运行中的任务日志被仪表盘误显示为失败的问题，`status=2` 现在会正确识别为运行中
- 修复最近执行任务中的“查看日志 / 重新运行”跳转无效问题
- 修复仪表盘快捷操作里的“新建任务 / 导入脚本”只跳页不打开目标操作的问题
- 修复 viewer 访问仪表盘时“查看详情”固定跳转管理员页面导致无效的问题
- 修复仪表盘首次进入重复请求 dashboard/info 的问题，避免重复加载与重复报错
- 修复任务统计卡“查看更多”实际只是刷新页面的问题

### 前后端数据契约与权限

- Dashboard 日志返回补齐任务类型与标签字段，前端显示的触发方式、环境标签不再误判
- 通知渠道新增并持久化真实发送/测试状态字段：
  - `today_send_count`
  - `today_send_date`
  - `last_test_at`
  - `last_test_status`
- 用户列表补齐真实的 2FA 启用状态，撤掉无效邮箱/备注展示
- 统一前后端密码规则为 `6-128` 位，避免前端校验与后端规则不一致
- 任务页与日志页进一步按角色控制按钮显示，减少 viewer 点击后端必失败的误操作

### 其他修复

- 修复今日/昨日统计和趋势统计在非 UTC 时区下的自然日边界问题
- 修复日志弹窗缺少图标导入导致的运行时警告与图标缺失问题
- 修复 Android runtime 安装链路未复用 refresh token 机制的问题
- 修复接口文档搜索后当前选中接口可能已经被过滤掉却未自动切换的问题
- 修复 Open API 页“本月请求 / 每日额度”等错误口径，改为真实的“今日调用数 / 每小时额度”
