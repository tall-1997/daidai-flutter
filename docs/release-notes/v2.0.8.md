## 行为变更

### 环境变量更新

上一版 v2.0.7 把环境变量的业务身份改成了 `(name, remarks)` 二元组 + 上面做 upsert，结果同名 + 相同备注的变量再次「新建」会静默覆盖旧值，多账号场景里只有最后一条能跑，反直觉。本版本把这条语义撤回，改为**纯 insert + 运行时按 name 合并**：

- `POST /api/v1/envs` 不再做任何 upsert 查找，每次提交都独立新增一行。同名 + 同备注可以有任意多条。
- `PUT /api/v1/envs/:id` 移除 `(name, remarks)` 撞对码的 409 拒绝，两行可以都叫 `KEEPER/stable`。
- 数据库层的 `idx_env_vars_name_remarks` 唯一索引在启动时**自动 DROP**（幂等），老库升级一次即可。
- 脚本运行时行为维持不变：调度器和 TaskExecutor 已经按 `name` 分组、相同 name 的 value 用 `&` 拼接（value 含 `&` 时自动升级为 `&&` 分隔并反斜杠转义）注入给脚本，所以脚本拿到的就是 `sfsyUrl=值1&值2&值3`，`split('&')` 得到账号数组。
- `POST /api/v1/envs/import` 的 `merge` 模式不变，继续按 `(name, remarks)` 合并——这是用户明确选择的"合并导入"意图，和手动新建按钮语义独立。
- 插件侧推荐的 `GET /envs?keyword=xxx → 匹配 remarks → 决定 POST 还是 PUT /envs/:id` 工作流保持可用，API 文档不变。原来依赖 POST 自动 upsert 的插件需要切到这个显式查再改的路径，否则每次刷新 token 都会新增一行。

## 修复

### 定时任务「日志」按钮弹窗会被内容撑大导致整页滚动

- 定时任务列表点「日志」打开的 `LogViewer` 弹窗，之前尺寸会随日志长度变化，日志多时撑满视口、日志少时塌陷成一条；滚动条还跑到了外层整页上，和同项目里执行日志页的详情弹窗体验不一致。
- 根因是 scoped CSS 里 `:deep(.log-viewer-dialog)` 选择器命不中目标——`LogViewer.vue` 的 template 根节点本身就是 `<el-dialog>` 且会 teleport 到 `<body>`，Vue 的 `data-v-xxx` 属性不会跟到 teleport 后的元素上，编译出来的 `[data-v-xxx] .log-viewer-dialog` 匹配空集，固定高度、`margin: auto` 等规则全都没应用。
- 解决方式：在 `LogViewer.vue` 里新增一个独立的**非 scoped** `<style>` 块专门处理 `.log-viewer-dialog` 根元素的样式，编译后是纯 `.log-viewer-dialog` 选择器，不依赖 scope 即可生效。固定 `height: 90vh`（mobile fullscreen 走 `is-fullscreen` 分支保持全屏），`margin: auto` 配合 Element Plus 的 `align-center` flex 容器实现垂直+水平居中，内部 `.viewer-log` 仍然 `flex:1 + overflow:auto` 承担日志滚动。
- 执行日志页 `logs/index.vue` 的详情弹窗同步修正：固定 `height: 88vh` + `margin: auto`，避免只写 `max-height` 导致内容少时弹窗塌陷、只写 `margin: 0 auto` 导致贴视口底部。
