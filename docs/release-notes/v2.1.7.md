# v2.1.7

## 版本概览

`v2.1.7` 聚焦面板使用细节体验：放宽 Magisk 模块对 Android 系统版本的限制、补齐 OpenAPI 一键备份与机器码独立接口、修复加密脚本日志被误截断、解决移动端环境变量页面滑动误触排序、清理桌面端 Tab 切换后整页可滚动等问题；同时把订阅拉库的 cron 注释解析推广到三种主流写法，并对暗黑模式下的色差做了集中补丁。

## OpenAPI 与权限

### 系统备份接入 OpenAPI
- 备份相关 7 个接口（`POST /api/system/backup`、`GET /api/system/backups`、上传 / 下载 / 删除 / 恢复 / 恢复进度）从仅管理员令牌可调用，扩展为同时支持具备 `backup` 权限的 OpenAPI 应用调用。
- 后端权限链改为 `OpenAPIAccess("backup") + RequireRole("admin")`，普通管理员账号 Token 仍可直接调用，OpenAPI 应用必须显式授予 `backup` 范围。
- 「Open API 管理」页面的权限范围多选增加「系统备份」选项；接口文档同步登记备份相关端点。

### 单条环境变量查询
- 新增 `GET /api/envs/:id` 接口，根据 ID 直接返回单个环境变量详情，避免 OpenAPI 应用按 keyword 反复查询匹配。
- 路由继承现有 `OpenAPIAccess("envs") + RequireRole("operator")`，开通 `envs` 权限的应用可直接调用。

### 机器码独立接口
- 新增 `GET /api/system/machine-code`，仅返回面板机器码（UUID v4 大写）。OpenAPI 应用具备 `system` 权限即可调用，外部脚本不再需要解析整份系统信息或读取数据库。

### Open API 列表显示 App Secret
- 桌面端「Open API 管理」表格新增「App Secret」列，默认掩码显示 `••••••••••••••••`。
- 点击「查看」会弹密码验证对话框；通过后显示明文 + 复制按钮，关闭对话框或刷新后再次显示需重新验证。
- 移动端卡片同步该体验，复用同一份密码验证逻辑。

## 脚本运行

### 加密脚本日志不再被误截断
- 移除单行 8192 字节硬截断；输出改用 `bufio.Reader` 直接按换行符分行读取，单行可达 MB 级。
- 加密脚本（base64 / 大段密文）输出完整结果，不再因为单行过长被截断为 `... [行内容过长已截断]` 后被脚本误判为失败。
- 总日志体量仍受 `max_log_content_size` 全局配置保护，超出会显示「日志已截断，超过最大大小限制」。

### 网络代理修改后实时生效
- 系统设置页修改「代理地址」后，立即作用于面板内置健康检查（`系统健康` 检查项）以及 bash / Go 等标准命令的执行环境。
- 此前 `systemHealthCheckHTTPClient` 是单例只在启动时读取代理；现在改为按需构建，每次健康检查都会读取最新 `proxy_url`。
- bash / Go 等标准命令在 `buildEnv` 末尾自动追加 `HTTP_PROXY / HTTPS_PROXY / ALL_PROXY` 等环境变量，与 Python / Node 已有行为对齐。

## 订阅拉库

### 三种 cron 注释样式自动适配
拉库时识别脚本顶部 cron 注释，已稳定支持以下三种写法：

- **JS 块注释 `<cron> <filename>`**
  ```
  /*
  价格保护
  55 11 * * * jd_OnceApply.js
   */
  ```
- **Python docstring `<cron> <filename>`**
  ```python
  '''
  new Env('豆子7天统计');
  8 8 29 2 * jd_beans_7days.py
  '''
  ```
- **青龙单行声明 `cron "EXPR" filename, tag:xxx`**（本次新增）
  ```
  cron "6 6 6 6 *" jd_CheckCK.js, tag:京东CK检测by-ccwav
  ```

### 防误识别
- cron 行后跟随的 filename 必须与脚本基名匹配（EqualFold），否则忽略，避免邻接脚本声明被错抓。
- docstring 中的中文说明、`filename = '../config/env.sh'` 等含 `=` 的代码片段不会被误识别为 cron。
- 新增 6 个回归测试覆盖三种样式 + 文件名 mismatch fallback + 真实 B 站脚本 docstring 场景。

## 移动端体验

### 环境变量拖拽手柄
- 移动端环境变量卡片新增专门拖拽手柄按钮（Rank 图标），仅从手柄长按 350ms 才能触发排序。
- 普通上下滑动列表不再被识别为排序手势，从根本上解决「下滑就乱序」的问题。
- 桌面端拖拽手感保持原有体验，全行可拖拽。

## 仪表盘性能

- 「快捷操作」按钮 hover 改为仅切换底色，不再触发整个 hero 横幅区域的重新合成。
- 顶部紫色横幅的 blur 装饰层提升为独立合成层（`transform: translateZ(0)`），与主内容隔离；横幅自身加 `contain: layout paint`，避免外部交互引发昂贵的重绘。

## 全局布局与暗黑模式

### Tab 切换后整页可滚动问题
- 桌面端 `html / body` 加 `overflow: hidden` 兜底，杜绝因 transition 切换页面期间的瞬时溢出引发整页滚动条。
- 路由容器 `.route-shell` 加 `position: relative + contain: layout style + min-width: 0`，并显式让 `.dd-scroll-page` 与 `.dd-fixed-page` 共享同一份 flex 约束。
- 页面切换动画从 `translateY(6px) → 0` 改为纯 `opacity` 渐变，移除会引发布局抖动的 transform。

### 暗黑模式色差补丁
- 表格头部背景 / 文字色统一走 `var(--el-fill-color-light)`，覆盖 `inline-style` 写死的 `#f8fafc`。
- 仪表盘紫色欢迎横幅 / 个人设置 hero 切换为深色渐变；统计卡片背景与边框走 `--el-bg-color-overlay` / `--el-border-color-darker`。
- 弹窗 header / footer 边框颜色在 dark 模式下不再发亮。

### 列表「显示全部」时的卡顿
- 环境变量、定时任务列表的 `GET /api/envs`、`GET /api/tasks` 接口新增 `all=1` 参数，服务端一次性返回全部条目（硬上限 5000），不再依赖前端循环分页。
- 环境变量页面选择「全部」时只发起一次接口调用，去掉了滚动期间的等待与卡顿。

## Magisk 模块

### 系统要求下放到 Android 7
- `customize.sh` 把 API 检查从 `≥ 26`（Android 8）放宽到 `≥ 24`（Android 7）。
- Android 7.x 安装时会追加提示，说明部分机型可能受 SELinux / 命名空间限制无法启动。
- README 系统要求章节同步更新：`Android 7.0 (API 24) 及以上`，Magisk 推荐 `v24+`，`v20.4–v23` 可装但缺少模块卡片一键更新。
