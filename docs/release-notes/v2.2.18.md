# v2.2.18

v2.2.18 是 v2.2.17 之后的维护版本，重点围绕用户反馈继续修复问题、优化使用体验，并补充必要的兼容性调整。

## 修复

- **Docker 镜像缺少 Python 3.10 / 3.11**：修复 v2.2.17 Docker 镜像只安装系统默认 `python3`，导致依赖管理中 Python 3.10 / 3.11 不可用的问题。新版 amd64 / arm64 Docker 镜像会内置 `python3.10`、`python3.11`、`python3.12` 版本化解释器，面板启动后会在后台为三个版本预热独立托管依赖环境。
- **旧 Python 托管环境目录命名不清晰**：v2.2.17 及更早版本的默认托管环境使用 `deps/python/venv`，部分用户实际对应 Python 3.11，部分用户对应 Python 3.12，容易误解。v2.2.18 会在启动时检测旧 `venv` 内真实 Python 小版本，并迁移为 `deps/python/3.10`、`deps/python/3.11` 或 `deps/python/3.12`，不再默认把旧 `venv` 视为 Python 3.12。
- **新建任务默认 Python 版本固定为 3.12**：修复依赖管理中默认 Python 版本已切换为 3.10 / 3.11 时，新建定时任务表单仍默认选择 Python 3.12 的问题。v2.2.18 起新建任务会跟随依赖管理的默认 Python 版本；已有任务保持原本配置，不会被批量改动。
- **日志背景图片超过 2MB 上传后不生效**：修复日志背景图片允许选择较大文件但保存后无效果的问题。上传较大背景图时，前端会自动压缩为适合配置保存的图片数据后再预览和保存，避免因 base64 过大导致配置未能稳定生效。
- **任务日志目录只显示任务 ID**：修复在面板外查看任务日志目录时只能看到 `task_xxx`、难以判断对应任务的问题。v2.2.18 起新生成的任务日志目录会保留任务 ID 并追加任务名称，任务名为空时回退为脚本名，例如 `task_12_签到任务`；旧版 `task_12` 目录仍会被面板正常读取和管理。
- **脚本启动偶发 `argument list too long`**：修复脚本调试、Shell 任务、Go 任务以及前置/后置脚本在环境变量体积较大时可能启动失败的问题。v2.2.18 起运行时环境变量会写入临时环境文件，由子进程启动后加载，不再把大体积变量直接塞进进程启动参数/环境中；Shell 脚本内部仍可读取完整变量，但不会默认把超大变量继续 `export` 给 `mktemp`、`sed`、`awk` 等子命令，避免子命令再次触发系统 `ARG_MAX` 限制。

## 优化

- **环境变量编辑弹窗体验**：加宽并加高新建/编辑环境变量弹窗，变量值输入区默认展示更多内容，适合编辑长 Cookie、Token、多行变量等内容；弹窗底部操作区保持固定，减少保存前反复滚动。

## 升级说明

- Docker 部署用户如需获得 Python 3.10 / 3.11 / 3.12 多版本解释器，需要重新拉取 v2.2.18 镜像并重建容器；面板启动后会异步预热 Python 3.10 / 3.11 / 3.12 的独立虚拟环境，不阻塞面板启动。
- 正常 Docker 更新流程会拉取新镜像并重建容器，例如执行 `docker compose pull && docker compose up -d`，或由 Watchtower 完成镜像更新与容器重建。完成后 amd64 / arm64 容器内应包含 `python3.10`、`python3.11`、`python3.12`。
- 如果更新后容器内仍只有原本的 `python3`，通常说明没有成功拉取到 v2.2.18 新镜像、容器没有完成重建，或仍在使用旧镜像标签启动容器；请先确认镜像版本与容器创建时间。
- 重建容器后可在容器内执行 `python3.10 --version`、`python3.11 --version`、`python3.12 --version` 验证解释器是否已经存在。
- v2.2.18 首次启动会自动整理 `deps/python` 目录：旧的 `venv` 会按实际解释器版本迁移到对应版本目录；v2.2.17 已创建的 `3.10/venv`、`3.11/venv` 等嵌套目录也会整理为同级版本目录。随后面板会在后台预热 `3.10`、`3.11`、`3.12` 三个托管环境；若某个解释器不可用，对应环境会在依赖管理中提示不可用。若目标版本目录已经存在，旧目录会被保留为带时间戳的备份，不会覆盖已有环境。
- 新建任务的 Python 版本默认值来自依赖管理中的“默认 Python 版本”。如果用户把默认版本设置为 Python 3.11，新建任务会默认选择 3.11；编辑已有任务时仍优先显示该任务原本保存的 Python 版本。
- 本次 v2.2.18 补丁包重新构建了同版本二进制与 Docker 镜像，用于覆盖早期 v2.2.18 包中 Shell 子命令仍可能继承超大环境变量的问题；已安装早期 v2.2.18 二进制的用户需要重新下载同版本安装包。
- 32 位 `linux/386` / `linux/arm/v7` 镜像暂时保留基础 `python3`，如需稳定使用 Python 3.10 / 3.11 / 3.12 多版本运行环境，建议使用 amd64 / arm64 设备或自行构建包含对应解释器的镜像。
