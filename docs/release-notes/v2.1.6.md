# v2.1.6

## 版本概览

`v2.1.6` 是一次更新链路、任务状态兜底和页面细节体验的综合修复版本。重点解决二进制包手动解压会覆盖 `config.yaml` 的升级痛点，同时补齐环境变量多分组、定时任务日志快捷查看、面板重启后的运行态清理，以及仪表盘和脚本管理页面的 UI 细节问题。

## 更新与部署

### 二进制后台更新

- 面板后台更新现在会自动识别 Docker / 二进制部署方式。
- Windows 单机版和 Linux 原生二进制部署会自动匹配 GitHub Release 中的 `daidai-windows-amd64.zip` 或 `daidai-linux-*.tar.gz`。
- 二进制更新会在后台下载、解压、替换程序与 `web/` 前端文件，然后自动重启面板。
- 二进制更新会跳过并保留 `config.yaml`、`Dumb-Panel/`、`data/`、`logs/`、`backups/`、`.env` 等本地配置与数据目录，避免直接解压覆盖服务器配置。
- 更新包解压增加路径安全校验，拦截 zip-slip / tar-slip 这类越界路径。
- Magisk 模块版仍通过 Magisk / KernelSU / APatch 管理器更新，面板内不会误走普通二进制更新。

### 更新状态展示

- 系统设置页的更新确认弹窗会根据部署方式显示不同文案。
- Docker 更新继续展示镜像、镜像源、容器重建流程。
- 二进制更新展示更新包、安装目录、目标程序，以及“下载更新包 / 解压校验 / 后台替换 / 等待上线”流程，不再错误显示“拉取镜像 / 辅助容器”。
- `ddp update` 和 `ddp check` 增加二进制部署识别，能显示更新包、安装目录和目标程序。

## 任务与日志

### 运行态兜底

- 面板关闭或重启时会尝试中断当前运行任务，并清理任务 PID 与运行态。
- 面板启动后会对异常遗留的运行中任务做状态修正，避免“实际没有进程，页面仍显示运行中”的问题。
- 手动任务、普通任务和 cron 任务的运行态恢复逻辑统一收口，减少重启后状态误判。

### 定时任务日志查看

- 定时任务页面支持点击任务名称快速打开该任务日志，用户不需要再先进入详情或日志列表查找。
- 修复日志弹层打开时原表格除任务名称列外整体右移的问题，避免弹窗出现后列表宽度被异常压缩。
- 任务表格、详情弹窗、日志查看器等组件补齐固定布局约束，减少页面整体被内部弹层或侧栏撑动。

## 环境变量

- 环境变量支持一个变量绑定多个分组。
- 分组筛选支持多选，可以同时查看多个分组下的变量。
- 新建、编辑、批量分组、导入和命令行 `ddp env` 都适配多分组字段。
- 后端兼容旧的单分组字段，迁移后会进行规范化去重，避免重复分组和空分组污染数据。
- 增加多分组筛选、批量分组和旧字段兼容的回归测试。

## 仪表盘与页面体验

- 仪表盘“系统资源”区域补回面板启动时长显示，并替换掉原网络指标。
- 优化“最近执行任务”列表的小标题与内容对齐，减少视觉错位。
- 定时任务、脚本管理等桌面端页面继续收紧固定页面布局，避免主页面出现不必要的整体上下滚动。
- 修复脚本管理页面连续点击多个脚本后侧边栏可能出现异常挤压或错位的问题。

## 文档与发行

- README 更新最新稳定版到 `v2.1.6`。
- Windows 单机版 README 改为推荐后台更新，并说明会保留本地配置与数据。
- 更新章节补充 Docker 与二进制两种后台更新方式的差异。

## 测试与验证

- 后端测试：`server/go test ./...`
- 前端构建：`web/npm run build`
- 差异检查：`git diff --check`
- 新增二进制 Release 资产匹配、安全解压路径校验、环境变量多分组、任务运行态恢复等回归覆盖。

## 升级说明

- 推送 Git tag `v2.1.6` 后会自动触发 GitHub Release 工作流。
- 二进制部署用户升级后，后续可优先使用面板后台更新；只有目录权限不足或 GitHub Release 网络不可达时才需要手动下载。
- Docker 用户仍建议确认部署时已挂载 `/var/run/docker.sock`，否则只能检查新版本，不能在面板内重建容器。
