# v2.0.0

## 更新内容

### 任务管理

- 任务日志弹窗默认改成从顶部开始看，自动滚动默认关闭；要追新日志时再手动勾上即可。
- 任务定时现在支持多条规则；原来的"选择定时规则"改成了"增加定时规则"，下面那块常用规则卡片去掉了，改成在"下次执行"旁边放"常用规则"按钮，点了再弹窗选。

### 脚本管理

- 脚本管理的"调试运行"弹窗加了"保存"按钮，现在可以先落盘再跑，不用来回退出编辑器保存。

### 环境变量

- 环境变量新建/编辑里的"分组"改成可直接输入，也支持点击后列出当前已有分组。

### 通知渠道

- 企业微信应用推送加了可选"反代基础地址"，可以直接填你自己的 Nginx 反代地址，不再只能写官方 qyapi 域名。
- WxPusher 渠道补了当前官方发送接口里的可选 url / verifyPayType 字段，渠道名称也更新成了 WxPusher / ClawBot(iLink)。
- Telegram 的"代理地址"改成真正独立代理，不再误当成 API Host；现在 api_host 和 proxy 是两条独立配置链，支持 http/https/socks5。

### Open API

- Open API 新建应用默认不限速，0 就是无限制；列表里的"调用次数"改成"今日调用"，按当天统计显示。

### 运行时 & 容器

- 默认 Alpine 镜像补了 gcompat / libc6-compat / libstdc++，用来提升加密脚本里 .so 的兼容性，解决这类 glibc 风格扩展在容器里直接加载报错的问题。
- Python 运行时新增 import 级自动依赖安装，通过 `__import__` 钩子实现；脚本运行中途缺依赖也能自动安装并重试导入，不用等脚本整段失败后再重跑。
- 系统更新的容器重建逻辑改成保留当前容器实际挂载的 volumes/mounts，避免用户自定义的数据目录在更新后被还原到默认路径。

### 备份与恢复

- 青龙备份导入后的拉库修了两处：已有脚本目录不再直接报"目录已存在"，会自动切换成可拉取的 Git 仓库目录；拉库日志弹窗也新增了"停止"按钮，卡住时可以直接终止。
