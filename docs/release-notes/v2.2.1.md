# v2.2.1

## 版本概览

`v2.2.1` 是一个针对备份与恢复链路的修复版本，重点解决“创建备份时所选内容没有正确进入备份包”、“导入旧备份后污染当前新面板运行态配置”以及由此引发的一系列恢复后观感异常问题。

## 重点修复

### 1. 创建备份时勾选内容不同步

- 修复「系统设置 → 备份恢复 → 创建备份」里勾选“配置项 / 定时任务 / 环境变量 / 订阅 / 日志 / 脚本 / 依赖记录”时，前端模型可能没有正确把勾选结果同步回父组件的问题。
- 旧实现使用深层字段直接修改，在 `defineModel` 场景下容易出现“界面看起来勾上了，但请求体没有同步更新”的情况。
- 现在改为显式整对象回写，避免备份请求拿到错误或空的选择状态。

### 2. 备份包 manifest 内容校验补强

- 新增后端回归测试，直接校验创建出来的 `.tgz` 备份包中 `manifest.json` 是否真的包含所选内容。
- 测试覆盖：
  - 定时任务
  - 环境变量
  - 系统配置（例如 `panel_title`）

### 3. 恢复旧备份不再污染当前新面板运行态

- 修复导入旧备份后，当前面板的运行态配置被旧备份一起覆盖的问题。
- 现在恢复过程中会跳过并保留以下当前运行态键：
  - `auto_update_last_checked_at`
  - `auto_update_pending_version`
  - `auto_update_pending_started_at`
- 同时在恢复前先快照当前值，恢复完成后再回填，避免因清空 `system_configs` 表而把当前面板运行时状态丢掉。

### 4. 核心业务数据恢复链路回归验证

- 新增测试验证恢复后这些内容确实会被备份包替换写回：
  - 定时任务
  - 环境变量
  - 普通业务系统配置
- 这样可以更早发现“恢复看似成功但核心数据没有真正落库”的问题。

## 测试与验证

- 关键回归测试通过：
  - `TestCreateBackupIncludesSelectedContentInArchive`
  - `TestRestoreBackupManifestReplacesCoreBusinessData`
  - `TestRestoreBackupManifestSkipsAutoUpdateRuntimeStateConfigs`

## 升级说明

- 推送 Git tag `v2.2.1` 后将自动触发 GitHub Release 工作流。
- 本次为修复型更新，不涉及新的部署步骤。
- 如果你之前创建过“内容为空”的备份包，建议在升级后重新创建一份新备份再使用。
