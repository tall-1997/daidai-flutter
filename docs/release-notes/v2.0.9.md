## 新增架构支持

### 32 位平台：linux/386 与 linux/arm/v7

面板现在多了两个 32 位目标平台的构建产物，老 x86 PC、瘦客户端、树莓派 2/3/Zero 2W、ARMv7 盒子/路由器/NAS 都能直接跑原生二进制（不再需要自己交叉编译）。

**GitHub Release 产物新增**：

- `daidai-linux-386.tar.gz`（Linux 32 位 x86 静态二进制 + 前端 + `config.yaml`）
- `daidai-linux-armv7.tar.gz`（Linux ARMv7 静态二进制 + 前端 + `config.yaml`）

**Docker 镜像 manifest 扩展**：

- `linzixuanzz/daidai-panel:latest`（Alpine 运行时）manifest list 从 `amd64 + arm64` 扩展到 **4 个平台：`linux/amd64` / `linux/arm64` / `linux/386` / `linux/arm/v7`**，`docker pull` 会自动选对应架构。
- `linzixuanzz/daidai-panel:debian`（Debian 运行时）扩展到 `linux/amd64` / `linux/arm64` / `linux/arm/v7`。Node.js 官方不再为 Debian `bookworm-slim` 提供 linux/386 manifest，所以 debian tag 不带 386；需要在 32 位 x86 机器上跑的用户请用 `:latest`（Alpine 版）。

### 实现要点

- `CGO_ENABLED=0` 全静态编译，依赖 `glebarez/sqlite`（纯 Go SQLite port），代码里无 64 位平台假设，Go 1.25 对本仓库 `linux/386` 与 `linux/arm/v7` 干净交叉编译通过。
- Dockerfile / Dockerfile.debian 引入 `ARG TARGETVARIANT`，`RUN` 阶段按 `v7 → GOARM=7 / v6 → GOARM=6` 映射，让 docker buildx 在 `linux/arm/v7` 下能产出正确的 ARMv7 二进制。
- `frontend-builder` 阶段固定为 `FROM --platform=$BUILDPLATFORM`，避免 docker buildx 在构建 32 位目标时尝试拉根本不存在的 `node:bookworm-slim` 386 manifest。

### 如何使用

**Docker 用户**：无需改动任何配置。

```bash
docker pull docker.1ms.run/linzixuanzz/daidai-panel:latest
# 在 32 位 x86 主机上会自动拉 linux/386，在 ARMv7 盒子上自动拉 linux/arm/v7
```

**直接下载二进制**：在 GitHub Release 页面按架构下载对应 `.tar.gz`，解包后执行 `./daidai-linux-386`（或 `armv7` / `amd64` / `arm64`）。
