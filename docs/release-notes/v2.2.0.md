# v2.2.0

## 版本概览

`v2.2.0` 主要聚焦面板日常可用性、日志观感、Docker 更新体验和暗黑模式一致性。本次重点修复了面板日志页自动刷新时的白屏闪烁、系统更新镜像源保存后未正确作用到实际拉取目标的问题，同时补齐了更安全的 Watchtower 自动更新方案，并更新了赞助名单源地址。

## 重点修复

### 1. 面板日志自动刷新白屏

- 修复「系统设置 → 面板日志」在开启自动刷新后每次都触发整卡 `loading` 蒙层、导致页面反复白屏闪一下的问题。
- 现在仅首次加载显示完整加载态，后续自动刷新改为轻量同步状态，不会再每隔几秒遮住整块日志区域。
- 日志卡片新增“同步中”状态标记，自动刷新时仍能感知到数据正在更新，但不会影响阅读。

### 2. Gin 访问日志输出美化

- 原先面板日志中的 Gin 访问日志格式过长，例如：
  - `[INFO] [GIN] 2026/05/11 - 23:14:59 | 200 | 8.576817ms | 116.162.227.223 | GET "/api/tasks?page=1&page_size=20"`
- 现在会自动压缩为更简洁的统一格式：
  - `[INFO] GET /api/tasks?page=1&page_size=20 -> 200 8.576817ms @116.162.227.223`
- 这样在面板日志页面、日志复制、日志下载和日常排查时都会更干净、更易扫读。

### 3. 系统更新镜像源未生效

- 修复在「系统设置 → 代理设置」里修改 `update_image_mirror` 并保存后，实际更新仍继续使用旧镜像源前缀（例如 `docker.1ms.run`）的问题。
- 新逻辑会在保持第三方自定义镜像不被误改的前提下，正确识别并重写呆呆面板官方镜像拉取目标。
- 适用于：
  - `linzixuanzz/daidai-panel:latest`
  - `docker.io/linzixuanzz/daidai-panel:latest`
  - 已带旧镜像加速前缀的官方镜像

### 4. 赞助名单源更新

- 默认赞助名单源已从旧地址迁移到：
  - `https://dumblist.linzixuan.top/`
- 同时放宽了返回结构解析逻辑，兼容：
  - `{"data": {...}}`
  - 直接顶层返回 `sponsors / count / total_amount / updated_at`
- 这样后续赞助源结构轻微调整时，面板也更不容易直接失效。

## 功能增强

### 1. Watchtower 自动更新方案完善

- 默认 Docker Compose 示例已升级为“面板容器 + Watchtower 容器”双容器方案。
- `docker.sock` 只挂给 Watchtower，不再默认直接暴露给面板业务容器。
- 新增更稳妥的生产版配置：
  - `docker-compose.watchtower.prod.yml`
  - `.env.watchtower.prod.example`
  - `.env.watchtower.prod`
- 生产版支持：
  - 固定维护窗口（默认每天凌晨 04:00）
  - 更新汇总通知
  - Push Plus 通过 Generic Webhook 接入
  - `--rolling-restart`
  - `healthcheck`

### 2. 暗黑模式界面一致性

- 补齐了暗黑模式下多个高可见组件的背景、边框与文字颜色：
  - 系统更新进度弹窗
  - 备份恢复进度弹窗
  - 赞助墙
  - 设置页相关卡片和深色补丁
- 修复原先浅色渐变/白底卡片在暗黑模式中显得突兀、不协调的问题。

## 测试与验证

- 关键链路回归验证通过：
  - 面板日志格式压缩逻辑
  - 赞助列表代理与 fallback
  - 用户会话撤销链路
  - SSH Key 管理权限限制
  - 系统更新镜像源解析关键路径
- `go test ./handler` 关键测试通过
- 说明：
  - 全量 `service` 测试在当前 Windows 环境仍可能受外部 `git` 子进程 PATH 可见性影响，与本次改动无直接关系

## 升级说明

- 推送 Git tag `v2.2.0` 后将自动触发 GitHub Release 工作流。
- Release 工作流会自动：
  - 构建 Linux / Windows 二进制
  - 上传 GitHub Release 附件
  - 构建 Docker 镜像
  - 构建 Magisk 模块与 `update.json`
- Docker 用户如果已经切到 Watchtower 方案，无需再把 `docker.sock` 直接挂给面板容器。
