# v2.0.3

## 修复内容

### 更新与运行时

- 修复面板内一键更新在 `docker.sock` 绑定存在时遗漏 Docker 数据卷的问题，更新后会同时保留 bind mount 和 named volume，避免重建容器后看起来像“数据丢失”。
- 一键更新现在会继续保留用户自定义的数据挂载路径和 `DATA_DIR`，并对等价挂载做去重，避免重复拼接到新的 `docker run` 参数。
- 修复 Debian 运行时系统资源页内存显示为 `0 B / 0 B` 的问题，Linux 内存采集改为直接读取 `/proc/meminfo`，不再依赖 `free` 命令。

### 环境变量与日志

- 环境变量页新增每页数量记忆，刷新后会保持用户选择的 `20 / 50 / 100`，并支持“全部”模式一次显示全部变量。
- 修复环境变量值复制按钮在部分浏览器或非安全上下文下提示成功但实际未复制的问题，补充了剪贴板降级兜底逻辑。
- 执行日志页搜索改为按任务名称匹配，批量删除入口和选中状态同步优化。

### 脚本与订阅

- 修复脚本调试运行在未保存内容时切到 `/tmp/daidai-debug` 执行，导致本地同目录模块被第三方同名包抢占的问题；现在临时调试文件会在原脚本目录执行，Python 导入优先级也会优先使用原目录。
- 上述调试链路修复同时覆盖 JS/TS 相对路径依赖场景，避免 `require('./sendNotify')`、`require('./local-helper')` 这类本地依赖在调试运行时失效。
- 订阅拉库自动添加任务时，cron 识别现已支持文档字符串首行写法，例如 `1 9 11 11 1 bili_task_get_cookie.py` 这类“cron + 文件名”格式。
