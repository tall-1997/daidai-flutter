## 修复

### 极验验证码加载失败无提示

后端 `validateGeeTest` 在 5 个错误分支（上游请求失败、响应读取失败、上游 5xx、非 200 状态码、JSON 解码失败）均增加 `log.Printf` 日志记录，方便排查验证码不显示的原因。

前端登录页在验证码 SDK 加载失败时，不再静默吞掉错误，改为通过 `ElMessage.error` 显示具体错误信息并更新状态提示文本。

### 脚本文件管理缺少"移动到根目录"

脚本文件树右键菜单新增「移动到根目录」选项（仅对子目录下的文件/文件夹显示）。点击后弹出确认框，调用后端 move 接口将文件移动到根目录，并自动刷新文件树。

### 日志查看器切换页面后自动滚回顶部

`LogViewer` 组件在浏览器标签页切换导致 SSE 重连时，不再重置 `autoScroll` 状态和触发 `scrollToTop`。`startStream()` 新增 `isReconnect` 参数，页面可见性恢复时以重连模式调用，保持用户当前滚动位置不变。

### 定时任务点停止后被禁用

根本原因：`ResolveTaskInactiveStatus` 在任务状态为 Running 时无条件检查调度器是否存在该任务的 Job，而手动触发的任务（无 cron 表达式）或通过「立即执行」触发的任务本身不会注册到调度器，导致 `HasJob()` 返回 false 从而被判定为已禁用。

修复：仅当任务配置了 cron 定时规则（`task.UsesCronSchedule()`）时才检查调度器状态，其他情况保持 Enabled 状态。

### 二进制文件自动探测前端目录

单机版运行时（无 Nginx 反代），Go 后端按优先级自动探测 `web/`、`dist/` 等前端静态资源目录，无需用户手动配置 `server.web_dir`。

## 新功能

### 订阅管理支持指定子目录拉取

订阅管理新增「指定子目录」字段（仅 git-repo 类型显示），支持通过 `git sparse-checkout`（cone 模式）仅检出仓库中的指定子目录，多个目录以逗号分隔。

- 后端：`Subscription` 模型新增 `SubPath` 字段；`pullGitRepoWithCallback` 在 fetch/clone 后、reset 前调用 `applySparseCheckout` 配置 sparse-checkout 规则
- 前端：订阅编辑表单新增输入框，附带使用说明提示
- 留空时拉取全部内容，行为与之前完全一致
