# v2.1.9

## 版本概览

`v2.1.9` 聚焦面板核心可用性与日常使用体验，重点修复了小屏幕下多个页面布局不自适应、脚本管理保存偶发失败、多段 Cron 任务无法重新启用、日志窗口比例不协调，以及 Docker 启动路径的运行时阻塞问题；同时补上了 Linux `systemd` 守护管理、面板日志分级查看和脚本执行失败的更明确提示。

## 重点修复

### 1. 全局页面自适应与弹窗布局

- 统一收敛多个页面的头部占位与固定区域布局，解决小屏幕下“按钮被挤到可视区外、必须往下滑才能保存 / 复制 / 运行”的问题。
- 调整脚本管理、环境变量、依赖管理、执行日志、任务日志等页面的可视高度计算方式，避免标题移除后内容区域错位。
- 优化任务日志弹窗与执行日志详情弹窗的宽高比例、最大高度和遮罩观感，桌面端与移动端显示都更协调。

### 2. 脚本管理保存失败链路加固

- 修复 Windows 环境下根目录脚本保存被误判为“路径穿越”的问题，原因是临时目录长短路径混用导致的路径归一化不一致。
- 收紧脚本路径校验逻辑，明确拦截 `..`、绝对路径和非法路径片段，避免复制 / 保存 / 移动时出现边界放行。
- 保存前会重新校验目标文件是否仍存在、是否是文本文件；保存失败提示现在会区分为：
  - 文件已被删除或移动
  - 当前路径其实是目录
  - 二进制文件不可在线保存
  - 路径非法
  - Docker / Linux 挂载目录写入失败
- 当 Node.js 脚本缺依赖后自动安装成功、但实际是 `ERR_REQUIRE_ESM` 这类 ESM / CommonJS 模块兼容问题时，日志中会追加明确的人话提示，不再只显示一大段 Node 原始报错。

### 3. 多段 Cron 任务启用失败

- 修复“一个任务配置多条 Cron 规则时，创建和编辑都正常，但禁用后重新启用会提示无效 Cron 表达式”的问题。
- 根因是启用接口仍按单条 Cron 做校验，而创建 / 编辑 / 调度器本身都已经支持多段规则。
- 本次已统一单个启用、批量启用和通用批量 `enable` 的后端校验逻辑，多段规则现在可以正常启用并注册到调度器。

## 功能增强

### 1. 面板日志分级与前端日志查看页

- `panel.log` 增加 `DEBUG / INFO / WARN / ERROR` 级别前缀。
- `/system/panel-log` 与 `ddp logs --level ...` 支持按级别筛选。
- 新增“系统设置 → 面板日志”查看卡片，支持：
  - 级别筛选
  - 关键字搜索
  - 最近 N 行
  - 自动刷新
  - 复制 / 下载
  - 只看更新日志 / 只看错误日志

### 2. Linux `systemd` 守护管理

- 新增 `ddp service install|uninstall|start|stop|restart|status`。
- 二进制更新流程会在 `systemd` 守护模式下自动先停服务、更新完成后再拉起，减少人工干预。

### 3. 脚本树与目录选择优化

- 脚本树、目录选择器和相关前端视图默认过滤 `node_modules`，避免超大依赖目录淹没正常脚本结构。

### 4. 多段 Cron 展示优化

- 任务列表桌面端、移动端卡片、任务详情弹窗统一改为新的多段 Cron 展示组件。
- 多条规则会按编号分组、逐条显示在独立代码块中，阅读和排查更直观。

## Docker 与运行时

### Docker 启动路径优化

- 去掉容器启动阶段阻塞式 `python3 -m venv` 初始化，避免首次启动或重启时前台等待过久。
- 改为运行时按需创建托管 Python 环境，并在后台异步预热常用 venv，减少进入登录页前的阻塞感。

## 测试与验证

- `npm run build` 通过
- Linux Docker 干净环境实测通过，验证内容包括：

  - 目录误保存 / 误打开

  - 多段 Cron 任务启用
