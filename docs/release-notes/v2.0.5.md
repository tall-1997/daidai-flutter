# v2.0.5

## 新功能

### AI 智能代码助手

- 新增 AI 代码助手功能，支持对脚本进行智能修改、修复、生成和解释，内置 Diff 对比视图和一键应用。
- 支持接入 OpenAI（Chat / Responses）、Anthropic Claude、Google Gemini 及自定义兼容接口，可在设置页配置多个 AI 提供商并自由切换。
- 支持流式生成（SSE），实时显示 AI 生成进度，可随时取消生成。
- 脚本编辑器右键菜单集成"AI 助手"入口，选中代码后可直接发起 AI 修改或解释。
- 新增独立 AI Code 页面，支持脱离脚本文件独立使用 AI 生成和调试代码。
- AI 生成支持补丁模式（Unified Diff），自动对比差异并高亮展示，支持尾部空白模糊匹配提升补丁成功率。
- 新增 Prompt 快捷预设标签，根据当前模式（修改/修复/生成）动态展示常用提示词，点击即可追加到输入框。
- Explain 模式结果使用 Markdown 语法高亮展示，提升可读性。
- 新增 AI Temperature 可配置项（0-2），可在设置页调整生成随机性。
- 所有 AI 提供商统一设置 max_tokens 为 16384，确保长脚本生成不截断。
- 当前脚本内容超过 500000 字符时自动拒绝请求，防止超大载荷。
- AI 设置页新增客户端表单校验，保存时对已部分配置的提供商给出缺失字段警告。
- 生成失败时显示重试按钮和错误详情，生成成功后可一键复制结果。

## 修复与优化

### 依赖自动安装

- 修复代码运行器（代码调试器）缺少自动安装依赖和重试机制的问题，现在与调试运行保持一致。
- 修复 Python 本地 `.so`/`.pyd` 扩展文件被误判为 pip 依赖并触发安装的问题，预执行和后执行检测均已增加本地模块识别。
- 自动安装依赖从单次重试升级为最多 5 轮循环重试，一个脚本缺多个依赖时可逐个安装直至全部满足。
- 每轮重试仅检测最新一轮输出中的缺失依赖，避免误匹配历史日志中已处理的包名。
- 已安装的包如果仍然报错，现在会输出明确提示（版本不兼容或内部依赖异常），而非静默停止或重复安装。
- 新增脚本提示型依赖检测：识别输出中 `npm install xxx` 和 `pip install xxx` 格式的提示并自动安装，覆盖脚本自行报错的场景。
- 清理代码运行器中已废弃的 `detectMissingDep` 函数及相关正则，统一使用 service 层检测逻辑。

### 代码运行器

- 修复代码运行器运行失败时输出面板无任何错误展示的问题，现在启动失败、轮询异常等情况都会在面板内显示具体错误信息。
- 代码运行器新增与调试面板一致的错误提示（红色 Alert）和"重新运行"按钮。
- 代码运行器自动安装流程中增加 `isStopped` 检查，用户手动停止后不再继续安装和重试。
