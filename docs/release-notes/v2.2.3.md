## 版本概览

本次更新主要聚焦脚本管理与备份体验，重点补齐 `.mjs` 脚本运行支持，优化“新建脚本即编辑”的交互流程，移除整套 AI Code 相关页面与接入逻辑，并为面板备份新增每天 / 每周 / 每月的定时备份能力。

## 重点更新

### 1. 补齐 `.mjs` 脚本运行支持

- 修复 Node.js ES Module 脚本在面板内无法正常调试和执行的问题。
- 现在以下场景都已支持 `.mjs`：
  - 脚本调试
  - 代码运行器
  - 定时任务命令 `task xxx/index.mjs`
  - 订阅脚本识别
  - 自动依赖安装识别
  - 备份脚本文件收集
- 同时脚本编辑器已把 `.mjs` 识别为 JavaScript 语法高亮，不再退回纯文本显示。

### 2. 新建脚本后直接进入可编辑状态

- 优化脚本管理页交互：新建文件成功后，会自动打开该脚本、进入编辑模式，并直接聚焦到编辑器。
- 现在不需要再额外点一次“编辑”，创建后即可直接输入内容。

### 3. 调整脚本管理页操作布局

- “添加任务”按钮已从更多菜单中移出。
- 它现在替换到了原来 AI 助手的位置，成为脚本页顶部的直接主操作。
- 页面中间空状态也同步调整为默认引导“新建一个脚本”，更符合当前实际使用流程。

### 4. 移除 AI Code 相关页面与接入

- 完整移除前后端 AI Code 相关功能，包括：
  - 独立 AI Code 页面
  - 脚本页 AI 助手入口
  - 前端路由、导航入口、配置面板
  - 后端 AI 接口、服务实现与相关测试
- 系统设置中不再保留 AI 提供商配置项，脚本管理页面默认回到纯脚本编辑 / 调试 / 任务管理工作流。

### 5. 新增定时备份能力

- 在“系统设置 → 备份恢复”中新增定时备份配置。
- 现在支持按以下频率自动创建面板备份：
  - 每天
  - 每周
  - 每月
- 可配置项包括：
  - 执行时间
  - 每周执行日
  - 每月执行日
  - 备份文件名前缀
  - 是否加密及加密密码
  - 备份包含内容（配置、任务、订阅、环境变量、日志、脚本、依赖）
- 保存配置后会立即刷新后端备份调度器，无需重启面板。

## 测试与验证

- 前端构建通过：
  - `npm --prefix web run build`
- 后端测试通过：
  - `go test ./handler ./service`
- 关键新增回归覆盖：
  - `.mjs` 脚本运行映射
  - `task xxx.mjs` 命令解析
  - 定时备份选择解析
  - 定时备份 Cron 生成

## 升级说明

- 推送对应 Git tag 后将自动触发 GitHub Release 工作流。
- 本次更新移除了 AI Code 相关页面与配置项；如果你之前已填写过 AI 提供商配置，这些配置将不再在面板中展示或使用。
- 如果你准备启用定时备份，升级后请进入“系统设置 → 备份恢复”保存一次定时配置。
