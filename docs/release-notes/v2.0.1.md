# v2.0.1

## 修复内容

### 运行时

- 修复 Python 自动依赖安装误触发的问题：第三方库内部的可选依赖探测（如 urllib3 尝试 import brotlicffi、backports 等）也会被 import 钩子拦截并尝试 pip install，导致大量无意义的安装尝试和报错。
- 自动依赖安装改为预扫描机制：执行脚本前通过 AST 静态分析提取用户脚本中直接编写的 import 语句，仅对确实缺失的依赖执行安装，不再干扰第三方库内部的模块加载逻辑。
- Go 端依赖检测新增 Python 标准库过滤，避免对内置模块（_winapi、winreg、nt 等）触发安装流程。
